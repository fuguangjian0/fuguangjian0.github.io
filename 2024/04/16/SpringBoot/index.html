为什么要用SpringBoot？

  1. 独立运行:springboot 内嵌了各种servlet容器，如Tomcat Jetty等，现在不需要打成war包部署到容器中，SpringBoot只要打成可执行的jar包即可独立运行，所有依赖包都在一个jar包中
  2. 简化配置：springboot提供了很多starter依赖，可以快速集成到其他框架和组件，大大简化了配置过程
  3. 自动配置：SpringBoot自动配置很强大，可以根据类路径下的jar包，类文件，注解来自动配置bean，减少了手动配置的工作量
  4. 无代码生成和xml配置：springboot使用条件注解和自动配置来完成所有配置工作，避免了代码生成和繁琐的xml配置文件，让开发更简洁，清晰
  5. 应用监控：SpringBoot提供很多有用的端点和指标，可以用来监控和管理应用程序，包括健康检查，性能监控，日志查询等，可以与Actuator集成，提供更丰富的监控功能

SpringBoot的核心注解是哪个？它主要由哪几个注解组成的

核心注解是@SpringBootApplication，由3个注解组成：

  1. @Configuration 声明当前类时一个配置类
  2. @EnableAutoConfiguration 开启springboot自动配置功能
  3. @ComponentScan 开启组件扫描，会自动扫描当前包及子包下的所有类，并注册为Spring的Bean
    这三个注解组合，让SpringBoot自动配置并启动应用程序，此外，@RestController，@Service，@Repository等，帮我们创建控制器，服务层，数据访问层等组件

运行SpringBoot应用程序有哪几种方式

  1. 在IDE运行，在集成开发环境IDE中，直接运行应用程序的主类来启动SpringBoot程序
  2. 打包成可执行jar包运行，使用maven插件可以打包jar文件，在命令行运行jar文件即可启动
  3. 在容器中运行：可以将应用程序部署到web容器，如Tomcat，Jetty等，然后启动容器即可运行应用程序，在部署时将应用程序打成war包，将war包部署到相应的容器中

如何理解SpringBoot中的starters

启动器（starter）是一种依赖管理方式，是一组预定义的依赖关系，将常用的依赖库和框架引入到项目中，使用starter可以避免在pom.xml引入依赖麻烦，检查依赖管理的过程。去steam创意工坊找mod，有一个常用mod合集

如何在SpringBoot启动时运行一些特定的代码

实现ApplicationRunner或CommandLineRunner接口来运行一些代码，是springboot的回调接口，作用：在应用程序启动完成后，自动回调run方法，在这个方法编写初始化代码，如加载配置文件，初始化缓存等
ApplicationRunner 接口的run方法传入ApplicationArguments对象，用来获取命令行参数和选项信息
CommandLineRunner接口的run方法传入String数组，获取命令行参数

实现两个接口，重写run方法，编写一些初始化代码，使用@Component注解表示是一个spring组件，可以被自动扫描并注册到spring容器，当springboot启动时，run方法自动回调，从而执行其中的代码

SpringBoot需要独立的容器运行吗

不用，boot应用程序已经被打包成jar包，可以独立完成，不需要外部容器，启动程序时，内嵌一个tomcat或jetty或undertow等servlet容器，可以直接运行web应用程序，不需要单独安装和配置外部servlet容器，称为嵌入式servlet容器
如果需要在生产环境部署springboot程序，可以打成war包，然后部署到servlet容器，如tomcat或jetty或websphere等，这种部署方式失去了springboot嵌入式容器的优势

SpringBoot中的监视器是什么

内置功能，让开发者监控应用程序的运行状况和性能指标，监视器基于SpringBootActuator模块实现，通过HTTP接口或JMX访问
监视器提供功能：

  1. 健康检查：通过访问/actuator/health接口，获取应用健康状况，如果应用正常运行，返回HTTP 200 OK响应，如果应用程序异常，返回一个HTTP 503 Service Unavailable响应
  2. 指标监控：通过访问/actuator/metrics接口，获取应用指标，如内存使用情况，CPU占用率，HTTP请求数等
  3. 环境信息：通过访问/actuator/env接口，获取应用环境信息，如操作系统，java版本，系统属性等
  4. 配置信息：通过访问/actuator/configprops接口，获取应用程序配置信息，包括SpringBoot自动配置的所有bean和属性
  5. 执行器端点：提供一系列执行器端点，访问应用程序内部状态和功能，如/actuator/beans /actuator/mappings /actuator/trace 等
  6. 监视器功能通过pom.xml引入spring-boot-starter-actuator依赖实现，该依赖自动配置Actuator模块，启用所有端点，通过配置文件或@Endpoint注解对端点进行自定义配置

如何使用SpringBoot实现异常处理

在SpringBoot中，可以使用@ControllerAdvice注解定义一个全局异常处理器，统一处理应用程序异常

  1. 定义异常处理类，使用@ControllerAdvice标注
  2. 在处理方法定义@ExceptionHandler注解，指定要处理的异常类型，可以使用多个@ExceptionHandler 注解定义不同的方法来处理不同的异常
  3. 处理异常，并返回合适的HTTP响应，可以返回一个自定义的错误信息，一个HTTP错误码，或者其他合适的响应

springboot中常用的starter？

  1. spring-boot-starter-web 用于构建web应用程序，包括springmvc tomcat容器等
  2. spring-boot-starter-data-jpa 用于使用jpa和hibernate进行持久化操作
  3. spring-boot-starter-data-security 用于构建安全的web应用程序，包括spring security OAuth2等
  4. spring-boot-starter-actuator 用于监视应用程序的运行状况和性能指标，包括健康检查，指标监控，环境信息，配置信息等
  5. spring-boot-starter-test 用于构建测试应用程序，包括JUnit Mockito Hamcrest等
  6. spring-boot-starter-log4j2 用于使用Log4j2进行日志记录
  7. spring-boot-starter-logging 用于使用SLF4j和Logback进行日志记录
  8. spirng-boot-starter-starter-amqp 使用AMQP进行消息队列和消息总线操作

SpringBoot实现热部署有哪几种方式？

  1. 使用springboot devtools提供了许多开发工具，包括自动重启，热部署功能，只需将springbootdevtools依赖添加到项目即可启用功能
  2. 使用springloaded是一个轻量级的java类热部署工具，可以让开发者不停止和重新启动应用程序的情况下更新java类，只需将spring-loaded依赖添加到项目即可启用功能
  3. 使用JRebel，JRebel是一款商业化的Java热部署工具，需要将JRebel插件继承到开发环境使用

如何理解SpringBoot配置加载顺序？

SpringBoot中配置文件的加载顺序时非常重要的，不同的加载顺序可以对应不同的配置场景。

  1. 命令行参数：动态修改配置，使用 –server.port=8080 来指定应用程序的端口号
  2. Java系统属性：使用-D参数设置Java系统属性，这些属性可以在springboot直接使用，可以使用-Dapp.version=1.0.0设置程序版本号
  3. OS环境变量：读取操作系统的环境变量，可以在应用程序直接使用，如：export APP_NAME=myapp设置应用程序名称
  4. application.properties或application.yml 文件，这些配置文件可以放在 src/main/resources 目录下或classpath根目录下
  5. 指定的配置文件：可以通过spring.config.name 和 spring.config.location 属性来指定要加载的配置文件，例如可以使用-spring.config.name=myconfig –spring.config.location=/etc/myapp/ 来加载指定的配置文件
  6. 通过@ConfigurationProperties 注解加载配置： 可以使用@ConfigurationProperties 注解将配置文件中的属性映射到Java类中

SpringBoot的核心配置文件有哪几个？他们的区别是什么

两个核心配置文件：

  1. application.yml 是springboot的主要配置文件，包含了应用程序各种配置属性，如数据库连接，端口号等
  2. bootstrap.yml 应用场景时当应用程序需要连接外部服务时，可以将连接属性配置在bootstrap中，以便在应用程序启动时能加载属性，这样可以避免将敏感信息暴露在配置文件，提高应用程序的安全性。
  3. 这些属性将在应用程序启动时加载，并用于与配置服务器进行通信，以获取应用程序的配置信息。由于这些属性是保存在bootstrap文件中，因此可以避免在应用程序配置文件中直接暴露这些属性。
