idea的一些使用技巧

Debug

4个箭头

  1. 恢复: 跳到下一个断点（如果有），没有直接运行到完成
  2. 步过： 执行下一行代码
  3. 步入： 执行当前行代码以及方法内部的代码，进入方法内部
  4. 步出： 跳出当前方法，继续执行后续代码

给方法打断点打在头和尾,方便定位这个方法

给ServiceImpl打断点,如果找不到ServiceImpl, 打在Service上

Git

冲突
|————-时间线————————————|
小明: |——-克隆小刚的代码—|—–修改代码 a.java—-|–提交代码(冲突)–|
小刚: |————修改代码 a.java——|-提交代码-|

点击合并(merge) -> 根据提示框操作

基本操作
找到左侧项目对应一个文件,右键选择 add commit rollback， git reset(重置HEAD)
先拉pull: 右上角箭头拉
后推push: 右下角分支名称处操作

提交点回滚
左下角git点开看到commit历史进度, 找到对应点右键 rollback, 弹窗选择hard(硬)

分支

快捷键

快捷键 功能
ctrl + f12 查看当前类的所有方法
ctrl + h 打开一个叫做“类型层次结构”的视图
ctrl + alt + m 抽取方法
ctrl + alt + o 优化导入的类,把import不用的删去
ctrl + alt + t try-catch
ctrl + y 删除当前行
ctrl + Shift + f 查找强化,everything
ctrl + Shift + j 自动将下一行上移到当前行末尾
ctrl + shift + z 取消撤销
F2 跳转到下一个高亮错误 或 警告位置

IDEA Windows + Mac 快捷键（全）
